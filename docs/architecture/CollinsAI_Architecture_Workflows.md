## Primary Flows

### Web contact form (fast path)

- Next.js posts to `POST /api/email` (serverless route). Route validates and forwards to `FastAPI /ingest/contact` or enqueues a job in n8n.
- FastAPI persists contact in `contacts` table and emits an event to PostHog.
- FastAPI calls Resend to deliver confirmation email.
- n8n workflow (optional) enriches lead, writes back to Postgres, notifies Slack/WhatsApp, and schedules follow-ups.

### Automation / operational task (n8n-driven)

- n8n triggers on inbound events (Postgres trigger, webhook, schedule).
- n8n executes actions: call external APIs (ERP, payment gateway), update DB, push notifications via WhatsApp gateway, and emit PostHog events.

## n8n workflow nodes (example)

- Webhook (receive contact)
- HTTP Request (validate/enrich)
- PostgreSQL (insert contact)
- Function (transform)
- HTTP Request (call WhatsApp / gateway)
- Wait / Cron (schedule follow-ups)

## Postgres schema (simplified)

- contacts(id PK, name, email, phone, source, payload jsonb, created_at)
- tasks(id PK, type, payload jsonb, status, run_at, created_at)
- events(id PK, subject, body jsonb, created_at)

## Security & Governance notes

- Store secrets in n8n credentials / cloud secret manager (do not commit keys).
- Use HTTPS, HSTS, CSP headers (already set in Next.js layout). Restrict incoming webhooks via HMAC signing.
- Enforce DB row-level permissions and audit logs for PII access.
